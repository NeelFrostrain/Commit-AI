# Contributing to Commit-AI

Thank you for wanting to contribute! Your help makes Commit-AI better for everyone.

## How to contribute

1. Fork the repository and create a feature branch:

```bash
git checkout -b feat/my-change
```

2. Make small, focused changes with clear commit messages. Follow the project's commit style (`type: description`).

3. Add tests where appropriate (see `Testing & recommended checks` in the README).

4. Run the test suite (if present) and linters:

```bash
# If tests are added later
npm test
# or run linter
npm run lint
```

5. Open a Pull Request with a clear description of the change and motivation.

## Reporting bugs

- Open an issue and include a short reproduction, expected behavior, and actual behavior.
- Include `node`/`bun` and OS versions if relevant.

## Feature requests

- Open an issue and explain the problem you want to solve and how you envision the feature.

## Code standards

- Follow existing code style (TypeScript, consistent formatting).
- Keep changes small and testable.

## Review process

- Pull requests will be reviewed and may get feedback before merging.
- Maintain a polite and constructive tone.
