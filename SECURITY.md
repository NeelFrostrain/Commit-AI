# Security Policy

If you discover a vulnerability, please follow these guidelines to report it responsibly.

## Reporting a Vulnerability

- Do not open a public issue for security-sensitive information.
- Instead, email the repository owner at: `security@your-domain.example` (replace with a real contact if you use this repo publicly).
- Include the following information:
  - A short description of the vulnerability
  - Steps to reproduce (minimal repro recommended)
  - Potential impact and suggested mitigation

## Timeline & Expectations

- The maintainers will acknowledge the report within a few business days.
- We will work with you to promptly triage and fix the issue, and issue a public advisory when resolved.

## Disclosure

- Coordinated disclosure is appreciated; please allow time for a fix before public disclosure.

Thank you for helping keep this project secure.
